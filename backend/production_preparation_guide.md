# Panduan Persiapan Produksi & Rilis (Production Readiness) - MEC System (MECS)

Dokumen ini berisi daftar cek (checklist) dan langkah-langkah teknis untuk mempersiapkan sistem **Military Equipment Control (MECS)** atau **Laboratory Information Management System (LIMS)** dari lingkungan development ke produksi.

---

## 1. Konfigurasi Lingkungan (.env)
Pastikan file `.env` di server produksi telah disesuaikan dengan nilai keamanan tinggi.

| Variable | Development | Production (Rekomendasi) |
| :--- | :--- | :--- |
| `GIN_MODE` | `debug` | `release` |
| `DB_DEBUG` | `true` | `silent` atau `false` |
| `DB_NAME` | `mec_db` | `mec_prod_db` |
| `SEED_DB` | `true` | `false` (Kecuali instalasi pertama) |
| `JWT_SECRET` | `your_super_secret_key` | **Ganti dengan string random panjang & aman** |
| `CAMUNDA_ENV` | `development` | `production` |
| `MINIO_ENV` | `development` | `production` |
| `MINIO_USE_SSL`| `false` | `true` (Jika menggunakan HTTPS) |
| `DB_SSLMODE` | `disable` | `require` atau `verify-full` |

> [!IMPORTANT]
> Jangan gunakan `localhost` di sisi client. Browser pengguna akan mengakses server melalui IP Publik/Internal atau Domain.

## 2. Arsitektur Deployment

### Opsi A: Unified (Go serves frontend)
Cocok untuk deployment sederhana. Backend Go melayani file statis frontend.
1. Jalankan `npm run build` di folder `frontend`.
2. Pastikan file ada di `frontend/dist`.
3. Go akan otomatis melayani `index.html` jika `GIN_MODE=release`.

### Opsi B: Separated (Nginx + Go) - DIREKOMENDASIKAN
Nginx menangani traffic port 80/443, melayani file statis, dan meneruskan API ke Go.

**Kelebihan:** 
* Performa servis file statis lebih baik.
* Manajemen SSL (HTTPS) tersentralisasi di Nginx.
* Keamanan (Header security, Rate limiting).

### Opsi C: Fully Decoupled (Berbeda Server/Port)
frontend dan backend berjalan sebagai dua entitas yang benar-benar terpisah.

1. **Konfigurasi frontend**:
   - Edit `frontend/.env.production` dan masukkan URL absolut backend pada `VITE_API_URL` (misal: `http://192.168.1.10:8081/api`).
   - Jalankan `npm run build`.
   - Sajikan folder `dist` menggunakan server statis (Nginx, Amazon S3, atau `npx serve`).

2. **Konfigurasi Backend**:
   - Pastikan Middleware CORS mengizinkan origin dari server frontend.
   - Jalankan backend Go di server/port tujuannya.

**Kelebihan:** 
* Skalabilitas tinggi (frontend dan backend bisa di-scale secara terpisah).
* Bisa menggunakan layanan hosting statis yang murah/gratis.

## 3. Konfigurasi Nginx (Contoh)
Simpan konfigurasi ini di `/etc/nginx/sites-available/mecs`.

```nginx
server {
    listen 80;
    server_name mecs.company.com; # Ganti dengan domain atau IP Server

    # Root folder hasil build frontend
    root /var/www/mecs/frontend/dist;
    index index.html;

    # Handle frontend Routes (React SPA)
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Proxy API requests ke Go Backend
    location /api {
        proxy_pass http://127.0.0.1:8081; # Port Go di .env
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Proxy Uploads (Opsional jika ingin dilayani Go)
    location /uploads {
        proxy_pass http://127.0.0.1:8081/uploads;
    }

    # Pengaturan Keamanan Upload (Maksimal size)
    client_max_body_size 50M;
}
```

## 6. Persiapan Service (Linux Systemd)
Buat file `/etc/systemd/system/mecs.service`:
```ini
[Unit]
Description=MEC System Backend
After=network.target postgresql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/mecs/backend
ExecStart=/var/www/mecs/backend/mec_prod
Restart=always
RestartSec=5
EnvironmentFile=/var/www/mecs/backend/.env

[Install]
WantedBy=multi-user.target
```
